# Changelog

## v0.3.5 Baseline - 2026-07-22

### Added
- Supabase 실제 회원용 거래, 거래방, 채팅과 비공개 첨부파일 저장 구조를 추가했습니다.
- 승인된 시공점만 딜러의 시공점 목록과 신규 요청 대상으로 조회하는 구조를 추가했습니다.
- 거래 참여자와 관리자 기준의 RLS 및 Realtime 기반을 추가했습니다.
- 데모 계정 `1/1`, `2/2`, `3/3`을 이용한 역할별 QA 흐름을 제공합니다.

### Changed
- 현재 개발 기준 버전을 `v0.3.5 Baseline`으로 통일했습니다.
- 실제 회원 데이터는 Supabase를 사용하고, 공개 데모 계정 데이터는 localStorage로 분리합니다.
- 가격 화면 명칭을 브랜드 기반 `권장 시공 패키지`로 정리했습니다.

### Fixed
- 공개 페이지가 인증 초기화 지연으로 무한 로딩되지 않도록 보완했습니다.
- 시공 요청 중복 생성과 채팅 메시지 중복 표시 가능성을 줄였습니다.
- 허용되지 않은 첨부파일과 과도한 파일 크기를 업로드 전에 검증합니다.

### Not included
- 결제·정산 실연동, 지도 API, 관리자 공지, 거래방 숨김·복원은 구현되지 않았습니다.

## V0.3.4.1 실제 회원가입 시스템 기반 - 2026-07-19

### Added
- Supabase 이메일 회원가입, 로그인, 로그아웃과 세션 복원 구조를 추가했습니다.
- 딜러·시공점 회원 DB 스키마와 RLS 정책을 추가했습니다.
- 시공점 승인 대기, 반려, 활동 정지 안내 화면을 추가했습니다.
- 관리자 시공점 가입 심사 패널을 추가했습니다.
- 보호 Route와 역할별 접근 정책을 추가했습니다.

### Changed
- Supabase 환경에서는 회원 프로필을 DB에서 불러오고 저장합니다.
- 신규 거래의 딜러 ID가 실제 로그인 회원 ID를 사용하도록 변경했습니다.
- Production에서는 개발용 데모 계정을 비활성화합니다.

### Preserved
- 거래, 거래방, 채팅, 자동 작업 브리핑과 결제 Prototype은 기존 localStorage 구조를 유지합니다.
- 개발 환경에서 Supabase가 설정되지 않은 경우 기존 데모 QA 흐름을 유지합니다.

## v0.3.3 시공점 워크스페이스 개선 패치 - 2026-07-18

### Added
- 오늘, 내일, 이번 주 단위의 시공점 작업 일정 UI를 추가했습니다.
- 메신저형 거래방에 이미지·일반 파일 첨부 Prototype과 이미지 미리보기를 추가했습니다.
- 실제 Cloud Storage로 교체 가능한 `AttachmentProvider` 경계를 추가했습니다.
- 시공점 관리에 휴무일과 긴급 시공 가능 여부를 추가했습니다.

### Changed
- 시공점 대시보드를 KPI보다 오늘의 작업을 우선하는 구조로 개선했습니다.
- 딜러와 시공점의 거래방을 거래 목록·대화·자동 작업 브리핑 3단 구조로 개선했습니다.
- 거래 채팅에 날짜 구분, 발신자, 시간, 반응형 상세 패널을 적용했습니다.

### Preserved
- 기존 거래, 거래방, 메시지, 자동 작업 브리핑과 상태 전이 로직을 유지했습니다.
- 첨부 파일 원본은 localStorage에 저장하지 않습니다.

## v0.3.2 API 연결 전 마지막 패치 - 2026-07-18

### Added
- 실제 인증 구현으로 교체할 수 있는 `AuthProvider`와 데모 인증 구현을 추가했습니다.
- `CurrentUser`, 표준 `UserRole`, 기존 `shop` 역할 호환 변환을 추가했습니다.
- 거래와 거래방을 `transactionId`로 조회하는 읽기 전용 서비스 경계를 추가했습니다.
- 향후 외부 서비스 연결에 필요한 환경변수 이름을 `.env.example`에 정리했습니다.

### Changed
- 로그인 화면의 계정 검증 책임을 UI에서 인증 서비스로 이동했습니다.
- 프로필 저장소에 명시적인 repository 인터페이스를 추가했습니다.
- 화면과 문서의 버전 표기를 V0.3.2로 통일했습니다.

### Preserved
- 데모 계정, 역할별 진입 화면, 가격 확인, 시공점 검색, 시공 요청, 거래방, 채팅과 결제 상태 흐름을 유지했습니다.
- 기존 localStorage 데이터 구조와 키를 변경하지 않아 별도 마이그레이션이 필요하지 않습니다.

## v0.2.5 UX validation update - 2026-07-16

### Changed
- `시공 가격 가이드` 화면을 딜러 업무 시작점인 `시공 가격 가이드` 화면으로 개편했습니다.
- 시공 가격 가이드에서 `견적 문의`와 `실제 시공 요청`을 분리해 같은 요청 구조로 이어지도록 개선했습니다.
- 시공 요청 화면의 추가 작업 입력을 자유 입력형 `추가 작업 문의`로 변경했습니다.
- 시공점 선택 카드를 별점, 동 단위 지역, 거리, 거래건수, 가능 작업 중심으로 정리했습니다.
- 거래방 상단에 자동 생성되는 `작업 브리핑` 카드를 추가했습니다.
- 거래 진행상태를 `접수`, `입고`, `시공중`, `완료` 버튼으로 변경할 수 있는 샘플 기능을 추가했습니다.
- 거래 내역을 `현재 진행`, `완료`, `취소` 탭 중심으로 정리했습니다.
- 시공점 화면을 `새 문의`, `진행중`, `완료` 업무 탭 중심으로 개선했습니다.
- 관리자 화면을 거래 수, 회원 수, 시공점 수, 오늘 거래 중심의 전체 현황 화면으로 단순화했습니다.
- 공통 카드, 버튼, 입력창, 모바일 반응형 스타일을 더 일관된 SaaS UI로 정리했습니다.
