# Car-Master

Car-Master는 리스·렌트 에이전시 딜러가 차량모델, 시공지역, 작업내용, 입고예정일 4가지만 입력해 전국 등록 시공점에 시공 요청을 보내고 거래를 관리하는 B2B 업무 플랫폼 프로토타입입니다.

현재 버전은 `Car-Master v0.2.5 - Dealer Price Guide & Package Selection`입니다.

## 현재 구현

- 딜러 중심 랜딩페이지
- 이메일/비밀번호 방식 데모 로그인
- 딜러 대시보드
- 딜러 금액표
- 브랜드별 신차패키지 샘플 상품 5종
- 차량 등급별 예상 금액 표시
- 금액표 상품 선택 후 시공 요청 화면으로 정보 전달
- 4개 항목 기반 내 시공 요청 흐름
- 요청정보 입력 → 시공점 선택 → 요청 확인 3단계 구조
- 샘플 등록 시공점 약 100곳
- 거래 상태 3단계: 시공점 확인중, 진행중, 작업완료
- 거래 관리 목록
- 수락 전 비활성화되는 거래 채팅 UI
- 시공점 요청 수락, 거절, 작업완료 처리 샘플
- 관리자 화면 유지
- 관리자 금액표 관리 샘플 UI

## 실행 방법

Node.js 22.13 이상이 필요합니다.

```bash
pnpm install
pnpm dev
```

브라우저에서 `http://localhost:3000`을 엽니다.

## 데모 계정

| 역할 | 아이디 | 비밀번호 |
| --- | --- | --- |
| 딜러 | `1` | `1` |
| 시공점 | `2` | `2` |
| 관리자 | `3` | `3` |

## 딜러 핵심 흐름

1. 로그인 후 대시보드에서 오늘 업무 상태 확인
2. `금액표`에서 브랜드와 상품 검색
3. 차량 등급별 예상 금액 확인
4. `이 상품으로 시공 요청` 클릭
5. `내 시공 요청`에서 선택 상품 요약 확인
6. 차량모델, 시공지역, 작업내용, 입고예정일 입력
7. 시공지역 주변 등록 시공점 목록 확인
8. 시공점 선택
9. 요청 내용 최종 확인
10. `시공 요청 보내기`
11. 거래 상태가 `시공점 확인중`으로 생성
12. 시공점 수락 후 거래 채팅 활성화
13. 작업완료 상태 확인

## 시공점 핵심 흐름

1. 새 요청 목록 확인
2. 차량모델, 시공지역, 작업내용, 입고예정일 확인
3. 요청 수락 또는 거절
4. 수락 후 거래 채팅 활성화
5. 작업완료 처리
