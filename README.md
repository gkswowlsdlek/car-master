# Car-Master

Car-Master는 리스·렌트 에이전시 딜러와 자동차 시공점을 연결하고, 가격 확인부터 시공 완료까지의 업무를 거래 단위로 관리하는 B2B 플랫폼입니다.

현재 버전은 `Car-Master V0.3.5.1 - 딜러 워크스페이스 UI/UX 개선 패치`입니다.

## 핵심 사용자 흐름

### 딜러

1. 시공 가격과 차량 등급별 가이드 금액 확인
2. 인도 지역 주변 시공점 검색 및 비교
3. 차량, 작업, 일정과 시공점을 선택해 요청 전송
4. 자동 생성된 거래방에서 일정, 채팅, 가격과 상태 확인

### 시공점

1. 대시보드에서 신규 요청과 오늘 입고 일정 확인
2. 우선 처리 거래와 다음 행동 확인
3. 거래방에서 작업 정보와 요청사항 확인
4. 입고예정, 입고, 시공중, 완료 단계 처리

### 관리자

1. 전체 거래와 확인 필요 상태 모니터링
2. 거래, 사용자, 시공점과 상태 검색
3. 결제 상태와 최근 채팅 기록 확인

## 실행 방법

Node.js 22.13 이상이 필요합니다.

```bash
pnpm install
pnpm dev
```

브라우저에서 `http://localhost:3000`을 엽니다.

## 베타 계정

| 역할 | 아이디 | 비밀번호 |
| --- | --- | --- |
| 딜러 | `1` | `1` |
| 시공점 | `2` | `2` |
| 관리자 | `3` | `3` |

## V0.3.4.1 실제 회원가입 시스템 기반

Supabase가 설정된 운영 환경에서는 이메일 인증과 회원 프로필을 Supabase Auth/Postgres에 저장합니다. 역할은 `dealer`, `installer`, `admin`으로 관리하며 시공점은 관리자 승인 후 워크스페이스에 접근합니다.

`shop`은 기존 화면과 저장 데이터 호환을 위해 유지되는 명칭이며, 새 서비스 경계에서는 표준 역할명 `installer`로 정규화합니다. 실제 API 연결 전에는 서버 인증, 권한 검증, 데이터베이스, 지도 API 키, 개인정보 처리방침과 운영 모니터링이 추가로 필요합니다.

거래, 거래방, 채팅과 결제 Prototype은 이번 버전에서 기존 localStorage를 유지합니다. 시험용 역할 계정은 공개 QA 기본값을 사용하며 서버 환경변수로 교체할 수 있고, 실제 이메일 회원 인증 및 Supabase 데이터 권한과 분리합니다.

외부 서비스 설정은 `SUPABASE_SETUP_V0.3.4.md`, 변경 사항은 `RELEASE_NOTES_V0.3.4.md`를 참고하세요.
