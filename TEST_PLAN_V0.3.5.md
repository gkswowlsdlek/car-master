# Car-Master v0.3.5 테스트 계획

## 자동 검증

1. `pnpm exec tsc --noEmit`
2. `pnpm lint`
3. `pnpm test`
4. `pnpm build`
5. `pnpm run build:vercel`

자동 테스트는 역할 접근, 인증 실패, 데모 세션 위변조, 거래 상태 전이, 수수료, 가격 브랜드 유지, 좌표 없는 시공점, SQL 핵심 정책, 메시지 정렬·중복 제거, 첨부파일 경계값, 손상 localStorage를 확인한다.

## 브라우저 회귀 테스트

- 320·360·390·768px 및 데스크톱에서 가로 스크롤과 버튼 겹침 확인.
- 딜러: 로그인 → 가격 확인 → 시공 요청 → 시공점 선택 → 요약 → 거래방.
- 시공점: 로그인 → 신규 요청 확인 → 거래방 → 단계 변경 → 채팅.
- 관리자: 로그인 → 가입 신청 목록 → 승인·반려·정지 UI.
- 로그인 실패, Supabase 미설정, 거래 로딩 실패, 빈 목록, 재시도 상태.
- 요청 전송 버튼 연속 클릭 시 한 번만 처리되는지 확인.
- 채팅 Enter 전송, Shift+Enter 줄바꿈, 긴 문장, 실패 후 재시도.
- JPG·PNG·WEBP·PDF·문서, 0바이트, 10MB 초과, 이중/대문자 확장자 확인.
- 새로고침·로그아웃·재로그인·다른 탭 로그아웃 확인.

## 실제 환경에서만 가능한 테스트

Supabase가 연결된 Preview에서 서로 다른 딜러·시공점 계정과 두 기기로 `SUPABASE_V0.3.6_CHECKLIST.md`를 수행한다.
