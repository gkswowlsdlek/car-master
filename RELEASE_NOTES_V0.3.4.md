# Car-Master V0.3.4 — 실제 회원 시스템 기반

V0.3.4는 데모 계정 중심 인증을 실제 이메일 회원가입과 회원 DB 구조로 전환하는 기반 버전입니다.

## 회원가입과 로그인

- 딜러와 시공점이 이메일과 비밀번호로 회원가입할 수 있습니다.
- 비밀번호는 애플리케이션 DB에 직접 저장하지 않고 Supabase Auth가 관리합니다.
- 로그인 세션은 쿠키 기반 SSR 구조로 새로고침 후에도 유지됩니다.
- Production 로그인 화면에는 데모 계정 정보를 표시하지 않습니다.

## 역할과 승인 정책

- 딜러는 가입 후 딜러 워크스페이스로 이동합니다.
- 시공점은 기본 `pending` 상태로 가입하고 승인 대기 화면으로 이동합니다.
- 관리자는 시공점 신청을 승인, 반려, 정지 또는 재심사 상태로 변경할 수 있습니다.
- 관리자 권한은 일반 회원가입이나 클라이언트 role 변경으로 생성할 수 없습니다.
- 역할과 승인 상태는 DB RLS와 Route 접근 정책에서 함께 확인합니다.

## 데이터 구분

- 회원 인증과 회원 프로필은 Supabase Auth/Postgres를 Source of Truth로 사용합니다.
- 기존 거래, 채팅, 결제 Prototype은 이번 버전에서 localStorage를 유지합니다.
- 신규 거래의 `dealerId`는 로그인한 실제 회원 ID를 사용하도록 준비했습니다.

## 외부 설정

코드에는 실제 Secret이 포함되어 있지 않습니다. Supabase 마이그레이션, Auth URL, Vercel 환경변수와 최초 관리자 지정은 `SUPABASE_SETUP_V0.3.4.md` 절차에 따라 사용자가 적용해야 합니다.
